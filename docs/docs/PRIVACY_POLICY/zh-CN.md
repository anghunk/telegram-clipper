# Clipper Hub 隐私政策

[English](./en.md) / [中文版](./zh-CN.md)

**最后更新：2025年12月23日**

本隐私政策描述了 Clipper Hub（"本扩展"、"我们"）在您使用浏览器扩展时如何处理信息。

## 概述

**Clipper Hub 的所有数据处理均在您的本地设备上完成。我们不会向我方服务器收集、存储或传输任何个人信息。本扩展仅将数据发送到您明确配置和授权的第三方平台（Telegram、Discord、Notion）。**

---

## 1. 我们访问的信息

为提供核心功能，本扩展可能访问以下数据：

### 1.1 网页内容
- **选中的文本**：您在网页上明确选择用于剪藏的文字
- **页面标题**：当前网页的标题
- **页面 URL**：当前网页的地址

### 1.2 用户配置
- 平台 API 令牌（Telegram Bot Token、Discord Webhook URL、Notion Integration Token）
- 数据库 ID 及其他平台特定设置
- 语言偏好

---

## 2. 我们如何使用信息

### 2.1 核心功能
所有访问的信息**仅用于**：
- 将您选中的内容发送到您配置的平台（Telegram、Discord、Notion）
- 在提交前的编辑界面中显示页面信息

### 2.2 仅限本地处理
- **所有数据处理均在您的设备本地进行**
- 仅当您主动点击"发送"或使用右键菜单时才会传输内容
- 数据**直接**从您的浏览器发送到您配置的平台（Telegram API、Discord Webhook、Notion API）

---

## 3. 数据存储

### 3.1 仅限本地存储
所有配置数据均使用浏览器内置的存储 API 在本地存储：
- `browser.storage.local` - 用于用户偏好和平台配置
- `browser.storage.sync` - 用于在您的设备间同步设置（通过您的浏览器账户，而非我方服务器）

### 3.2 本地存储的内容
- 平台启用/禁用状态
- 您提供的 API 令牌和凭据
- 语言偏好

### 3.3 我们不存储的内容
- 浏览历史
- 网页内容
- 选中的文本（仅在内存中处理，不会持久化存储）
- 分析或追踪数据

---

## 4. 数据共享

### 4.1 第三方平台
当您使用本扩展剪藏内容时，数据会**直接**发送到：
- **Telegram** (api.telegram.org) - 如果您已配置 Telegram
- **Discord** (discord.com/api/webhooks) - 如果您已配置 Discord
- **Notion** (api.notion.com) - 如果您已配置 Notion

这些传输仅在以下情况发生：
- 您明确发起发送操作时
- 直接从您的浏览器发送到平台的 API
- 使用您提供的凭据

### 4.2 无其他数据共享
- 我们**不运营**任何后端服务器
- 我们**不收集**分析或使用数据
- 我们**不与**广告商或数据代理商共享任何数据
- 我们**不使用**任何第三方追踪服务

---

## 5. 数据安全

- 所有 API 通信均使用 HTTPS 加密
- 您的凭据仅存储在浏览器的安全存储中
- 没有任何数据传输到我们控制的服务器

---

## 6. 您的权利和控制

您对您的数据拥有完全控制权：

### 6.1 访问和修改
- 通过扩展的设置页面查看和修改所有设置
- 所有存储的数据均可通过浏览器开发者工具访问

### 6.2 删除
- 卸载扩展会删除所有本地存储的数据
- 您可以通过浏览器设置手动清除扩展数据

### 6.3 退出
- 您可以随时禁用任何平台
- 如果没有您的明确配置，扩展将无法运行

---

## 7. 权限说明

| 权限 | 用途 |
|------|------|
| `contextMenus` | 创建右键菜单用于快速剪藏 |
| `storage` | 在本地存储您的设置 |
| `notifications` | 显示发送成功/失败通知 |
| `scripting` | 读取网页选中的文本 |
| `activeTab` | 获取当前标签页的标题和 URL |
| 平台 API 的主机权限 | 向您配置的平台发送内容 |

---

## 8. 儿童隐私

本扩展不面向 13 岁以下的儿童。我们不会故意收集儿童的信息。
